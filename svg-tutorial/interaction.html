如何使用JavaScript为SVG元素添加交互

在本章中，我们介绍了一个可以切换开关的灯泡和一个可以用鼠标拖动的可调灯具。在这两种情况下，我们为SVG元素分配事件监听器，使它们具有交互性。

示例 1：切换灯光

让我们从一个简单的例子开始。通过点击灯泡，我们可以切换灯光的开关。对于这个例子，我们将重用来自弧形课程的 灯泡 示例。以下是它的最终源代码。本章重点讲解交互，如果你需要回顾如何创建这个图标，请查看弧形章节。

然后，在JavaScript中，我们可以为这个元素分配一个事件处理程序，并切换它的填充颜色。我们通过 lightOn 变量跟踪灯光是开着还是关着，根据这个状态，我们将 fill 属性设置为金色或透明。

在CSS中，我们可以为灯泡添加一个指针样式的光标，以表示它是可点击的。

在React中切换灯光

我们可以使用React组件实现相同的逻辑。我们可以创建一个新的 lightOn 状态，并根据这个状态设置第一个路径元素的填充属性。然后，我们还可以通过绑定到同一个路径元素的 toggleLight 事件处理程序来切换灯光。

在使用React时，我们需要将某些属性名转换为camelCase。例如，我们使用 strokeWidth 代替 stroke-width，并使用 strokeLinejoin 代替 stroke-linejoin。

示例 2：调节灯具

现在，让我们来看一个更复杂的例子。你可以通过拖动灯头来调节灯具。你可以通过垂直拖动调节灯具的高度，通过水平拖动调节灯具的倾斜角度。

如何变换SVG元素

在这个例子中，我们重用了在前一章节中创建的 灯具。这是我们最终得到的源代码。这个SVG比较复杂，使用了变换来定位其元素。如果你需要回顾，请参阅变换章节。

上面的例子中唯一的区别是，一些组元素现在有了 id。我们使用这些 id 在 JavaScript 中访问这些组。

这个 SVG 的构造方式允许我们通过简单地调整一些旋转值来倾斜灯具的手臂。你可以看到，第一只手臂向左旋转了 25°，第二只手臂向右旋转了 50°，而连接灯头的第三只手臂再次向右倾斜了 45°。

这些就是我们在拖动灯头时需要更改的旋转值。因此，每个组都有一个 id (arm1、arm2、arm3)。我们可以通过 id 在 JavaScript 中获取这些组。

假设第二只手臂总是与第一只手臂平衡。如果第一只手臂向左旋转了 25°，第二只手臂将向右旋转 50°。它的旋转值总是第一只手臂旋转值的两倍，且方向相反。

因此，在代码中，我们只需要跟踪两个旋转值——第一只和第三只手臂的旋转。我们将这些值命名为 arm1Rotation 和 arm3Rotation。在 JavaScript 中，我们将这些变量初始化为与 SVG 中定义的旋转值相匹配的值。

然后，我们通过在灯头元素上添加 mousedown 事件监听器，在文档上添加 mousemove 和 mouseup 事件监听器，实现一个简单的拖拽功能。

我们通过 isDragging 变量跟踪鼠标是否在拖动灯头。在拖动的情况下，我们计算鼠标的水平和垂直移动距离。

从 dx 和 dy 中计算出第一只和第三只手臂的新旋转角度。我们根据垂直移动来改变第一只手臂的旋转角度，根据水平移动来改变第三只手臂的旋转角度。同时，我们限制这些旋转的范围。第一只手臂可以在 -60° 到 0° 之间倾斜，而第三只手臂可以在 -90° 到 90° 之间倾斜。

在每一步，我们都使用新的旋转值设置这些组的 transform 属性。然后，一旦鼠标按钮被释放，我们就更新 arm1Rotation 和 arm3Rotation，以保持灯具的当前状态。

最后，我们可以将灯头元素的鼠标光标改为“拖拽”状态，以指示我们可以调整灯具。

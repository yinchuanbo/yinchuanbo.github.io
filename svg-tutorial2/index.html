SVG 课程

本文是一个关于编写 SVG 文档的互动教程。它介绍了 SVG 规范的基本语法、结构和元素。教程使用 这个库使页面上的 SVG 元素具有互动性。

简介

SVG 是“可缩放矢量图形”（Scalable Vector Graphics）的缩写。SVG 文件类型和规范是一种基于 XML 的语言，用于创建矢量图形，可以作为独立文档或嵌入式文档使用。由于文件大小小且生成的图像清晰，SVG 是开发人员和艺术家创建基于矢量的图像（如徽标、图案和遮罩效果）时的热门选择。

SVG 文档中的元素使用 XML 语法定义，构成一个类似树状结构的对象模型，称为 SVG 文档对象模型（SVGDOM）。例如，下面的 SVG 绘制了三个等间距的圆形。svg 标签是文档的根元素，它定义了一个宽度为 200 和高度为 100 的绘图区域。每个圆形由 circle 标签描述，标签具有定义圆心位置和半径的属性。 

<svg width="200" height="100">
    <circle cx="50" cy="50" r="15"></circle>
    <circle cx="100" cy="50" r="15"></circle>
    <circle cx="150" cy="50" r="15"></circle>
</svg>

这个 SVG 在下方渲染

默认情况下，SVG坐标系统将原点放置在绘图区域的左上角。正的 x 方向指向右侧，正的 y 方向指向下方。这对于初学者来说可能有些复杂，但在计算机图形学中，通常会将坐标系统的 y 轴方向定义为向下，而不是向上。

XML 语法

构成SVG语言的XML元素是由开标签和闭标签组成的。标签名称指示正在定义的元素。开标签包含描述元素属性的属性。

例如，在下面的代码片段中，circle 标签定义了一个圆，圆心位于 (50,50) 点，半径为 15。属性 cx="50"、cy="50" 和 r="15" 定义了圆的位置和半径。 

<circle cx="50" cy="50" r="15"></circle>

除了元素特定的属性，还有一些全局属性可以为任何元素定义。例如，下面的组元素被赋予了 id example-id 和类 example-class。 

<g id="example-id" class="example-class">
  ...
</g>

全局属性对于样式设置、操作和查询元素非常有用。通常，id 属性是唯一的，用于标识单个元素。class 属性通常应用于一个或多个元素，以便以相同的方式对它们进行样式设置。

class 由空格分隔的类名列表

基本元素

基本视觉元素定义了形状和线条。例如，一些基本元素包括线条、圆形和椭圆。元素的几何属性通过其开标签中的属性来定义。注意，下面的元素是可交互的。点击并拖动任意一个点，查看元素如何绘制。

线条元素

线条元素在两个点 (x1, y1) 和 (x2, y2) 之间绘制一条线。样式属性 stroke 和 stroke-width 控制线条元素的颜色和宽度。

椭圆元素

椭圆元素绘制一个以点 (cx, cy) 为中心的椭圆，水平半径为 rx，垂直半径为 ry。椭圆的路径可以使用 CSS 的 stroke 和 stroke-width 属性进行样式设置，这些属性控制颜色和宽度。椭圆的填充颜色可以通过 CSS 的 fill 属性进行样式设置。

矩形元素

矩形元素绘制一个矩形，其左上角由点 (x, y) 定义，尺寸为 width 和 height。如果提供了负值的宽度或高度，矩形元素将不会被渲染。矩形的路径可以使用 CSS 的 stroke 和 stroke-width 属性进行样式设置，这些属性控制颜色和宽度。矩形的填充颜色可以通过 CSS 的 fill 属性进行样式设置。

路径元素

路径元素功能强大且复杂。到目前为止，所有元素都可以使用路径元素绘制。路径是通过一系列命令定义的。每个命令以一个字母开始，后跟一系列值，这些值描述了路径段的形状。以下是总结基本路径命令的表格。

命令 语法
Move To 
M x y
Line 
L x1 y1
Quadratic Bezier 
Q x1 y1 x2 y2
Cubic Bezier 
C x1 y1 x2 y2 x3 y3
Arc 
A rx ry rotation arc-flag sweep-flag x y
Close 
Z

命令可以使用绝对或相对坐标来定义。使用大写字母定义的命令表示绝对坐标，坐标是相对于坐标系的原点来定义的。使用小写字母定义的命令表示相对坐标，坐标是相对于路径的上一个位置来描述的。

路径移动和直线命令

<path d="M x y L x1 y1"></path>

大多数路径从“移动到”命令开始，该命令定义了路径在坐标系统中的起始位置。在下面的示例中，命令 M x y 会在点 (x,y) 处开始绘制路径。如果没有“移动到”命令，默认的起始点是 (0,0)。然后，由于“移动到”命令后面跟着字母 L，路径会绘制到由直线命令描述的点。请参见下面的示例。

路径二次贝塞尔曲线

<path d="Q x1 y1 x2 y2"></path>

SVG 规范提供了几种贝塞尔曲线的变体。下面显示的二次贝塞尔曲线命令使用路径的前一个位置和命令中定义的两个点来绘制二次贝塞尔曲线。

路径三次贝塞尔曲线

<path d="C x1 y1 x2 y2 x3 y3"></path>

三次贝塞尔曲线使用路径的前一个位置和命令中定义的三个点来绘制三次贝塞尔曲线。

路径弧命令

<path d="A rx ry rotation arc-flag sweep-flag x y"></path>

弧命令在两个点之间绘制一个弧线，使用由水平和垂直半径定义的椭圆曲线。如果两个点之间的距离大于提供的椭圆所能跨越的距离，则使用最小的椭圆来跨越该距离。旋转值定义了椭圆在空间中的方向，两个标志决定了绘制哪种类型的弧线。

请注意，由于涉及到计算椭圆在两个点之间的数学问题，存在两个解，从而产生两个可能的椭圆。

路径闭合命令

闭合命令“完成闭环”，将路径的当前位置连接到路径的起始点。这在绘制具有填充颜色的形状时非常有用。

路径元素是 SVG 规范的核心，且较为复杂。有关更多信息，请参阅 MDN 路径教程。

树形结构

SVG 文档形成了类似树形的结构，因为该语言基于 XML。每个元素包含零个或多个子元素，这些子元素被放置在元素的开始和结束标签之间。以下是一个简单的 SVG 示例。 

<svg width="300" height="200">
  <g>
    <circle cx="50" cy="50" r="10"></circle>
    <circle cx="100" cy="50" r="10"></circle>
  </g>
  <rect x="50" y="80" width="50" height="20"></rect>
</svg>

对应的树形结构如下所示。关于这个结构，有两点重要内容需要了解。1）它是层级性的，元素会继承应用于其父元素的变换和样式。例如，如果组元素应用了变换，它的两个子元素也会被变换。2）元素绘制的顺序取决于它们在文档中的顺序。先绘制的元素会出现在背景中，而最后绘制的元素会出现在前景中。

坐标系统

SVG 坐标系统的原点位于图像的左上角。正 x 方向是向右，正 y 方向是向下。在计算机图形学中，y 轴通常是翻转的，因为元素的位置是相对于容器的左上角来定义的。试着点击并拖动下面的控制点。

样式

SVG 元素使用层叠样式表或 CSS 进行样式化。 样式可以被应用于单个元素作为内联样式或使用用户定义的样式表。 许多演示属性可以设置样式,但最常见的是 fill 和 stroke。

内联样式

可以使用 style 属性将样式应用于单个元素。这对于测试不同的元素样式非常有用。 在实践中,一个良好构建的样式表将更加简洁,并减小 SVG 文件的大小。 下面显示了三个使用内联样式设置的圆圈。 

<svg xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="75" r="25" style="fill:#de4850"></circle>
  <circle cx="125" cy="75" r="25" style="fill:#eeb535;"></circle>
  <circle cx="200" cy="75" r="25" style="fill:#68c541;"></circle>
</svg>

样式表

内联样式的替代方案是样式表。样式表可以在 SVG style 元素中定义, 也可以作为外部样式表。样式表包含逻辑选择器,根据元素的类型、类和其他属性来设置样式。 选择器有优先级,可以包含复杂的逻辑来选择特定的元素。 

/* style elements by tag name */
circle {
  stroke: #d0d0d0; /* light grey */
  stroke-width: 4px;
}

/* style by user interaction */
circle:hover {
  stroke: #808080; /* medium grey */
}

/* style elements by class */
.red {
  fill: #de4850;
}
.yellow {
  fill: #eeb535;
}
.green {
  fill: #68c541;
}

<circle class="red" cx="50" cy="75" r="25"></circle>
<circle class="yellow" cx="125" cy="75" r="25"></circle>
<circle class="green" cx="200" cy="75" r="25"></circle>

排版

有许多元素可用于在 SVG 文档内创建和格式化文本。 请注意,在 SVG 中使用字体可能有些棘手。两个好的策略是 1) 使用 Web 安全字体, 这些字体可能存在于每个操作系统上,以及 2) 使用 Google 字体等 Web 字体。 虽然选项 2) 有更广泛的字体选择,但如果 SVG 是使用 HTML img 标签嵌入的,浏览器无法访问外部链接资源,因此会使用回退字体。

文本元素在文档的 (x, y) 点绘制文本。

高级元素

SVG 规范包含许多元素,虽然并非全部都在本教程中涉及,但有几个值得一提的高级元素。

剪裁路径元素

可以将剪裁路径应用于元素,只显示剪裁路径形状内包含的部分图形。通过将剪裁路径属性设置为指向 DOM 树中的剪裁路径元素来对元素及其子元素应用剪裁。例如,由子路径元素定义的剪裁路径在应用于图像时会产生 遮罩 效果。 

<clipPath id="my-clip-path">
  <path d="M 50 50 L 250 50 L 250 250 L 50 250 Z"/>
</clipPath>
<image clip-path="url(#my-clip-path)" href="..."/>

点击并拖动控制剪裁路径形状的点。

剪裁路径可以应用于更多元素,并且可以使用其他几何元素来定义。

Defs 元素

defs 元素包含一个可在文档中重用的图形"定义"。例如,可以定义一个箭头,然后将其附加到路径上。 

<svg>
  <defs>
    <!-- Defines the shape of the arrow-->
    <marker></marker>
  </defs>
  <!-- Uses the marker -->
  <path></path>
</svg>

这里没有展示更多高级元素。请查看 MDN 上记录的完整 SVG 元素列表 链接。

Web 使用

由于其响应式和可编程属性,SVG 文档是 Web 上最常见的使用案例之一。这一部分解释了如何在 Web 上 使用 SVG,并权衡了不同用例的优缺点。

嵌入 SVG

在 Web 上嵌入 SVG 文档有两种常见策略:使用 img 标签和作为内联 xml。根据您的使用情况,两种方式都有优点。

图像标签

<img src="/path/to/file.svg" width="400" height="300">

img 标签对于出现在多个地方的 SVG 文档很有用,并利用浏览器缓存,如果图像出现在用户导航的 许多页面上,这可能很有用。使用 img 标签时需要考虑的因素是:外部样式表不会影响 SVG 的样式, 并且无法使用 JavaScript 操作文档。

内联 SVG

<svg width="400" height="300">
  ...
</svg>

嵌入内联 SVG 文档的优点是它可以节省浏览器的资源请求,允许 JavaScript 与文档交互,并允许 对 SVG 元素进行更灵活的样式设置。这种方法通常用于图标和交互性。

内联 SVG 的缺点是可能会导致增加 html 大小,并且由于浏览器无法缓存图像, 后续页面加载不会比首次加载更快。另一个缺点是,如果内联的 SVG 在 style 元素中 定义了不同的选择器,可能会产生样式冲突。
